# 百萬流量求生記 — 系統設計公開課講義

跟著小電商「好物市集」從每天 500 人長到百萬流量:**一次危機,學一套武器**。
這裡是課程的完整講義與 lab 程式碼;搭配的對話式影音課在 YouTube 頻道
[教練與小艾](https://www.youtube.com/@coach-and-xiaoai)。

## 怎麼用這門課

1. 🎧 **先聽**:每個模組一支合輯影片(教練與小艾的對話),第一次接觸用聽的就好,程式碼不用記。
2. 💻 **再動手**:回到電腦,照講義把該模組的 lab 跑起來(需要 [Docker](https://www.docker.com/);每課都有可直接貼上的指令與程式碼)。
3. ✅ **自我驗收**:每課結尾有小挑戰與收尾一問;模組末有放手課與不看筆記小考。

## 課程地圖(4 個模組,37 課)

| 模組 | 危機 | 學到的武器 |
|---|---|---|
| [模組一:讀取危機](01_read-crisis/README.md) | 爆紅那晚,網站掛了 | 索引、快取、讀取副本、CDN 與負載平衡 |
| [模組二:寫入危機](02_write-crisis/README.md) | 一萬個人同時按下購買 | 交易與鎖、分散式鎖、佇列、分片、一致性雜湊、CAP |
| [模組三:功能危機](03_feature-crisis/README.md) | 需求雪崩:五張便利貼 | API 設計、大檔案、搜尋、即時更新、長時任務、資料管線、RAG |
| [模組四:半夜三點的告警](04_reliability/README.md) | 一次連鎖故障復盤 | 可觀測性、過載保護、可靠傳遞、container/serverless |

建議按順序上;每課開頭都標了「需要先會」。
