# L4|即時更新:到貨了要馬上知道 📖

🎯 這課結束時:面對「使用者要即時看到變化」這類需求,你能在 polling、SSE、WebSocket 之間做出有理由的選擇。
🧩 需要先會:無,把這課當成「三個工具怎麼選」的決策課。
📚 想深挖:MDN「Server-sent events」「WebSockets API」;關鍵字:long polling、EventSource、persistent connection。

## 熟客的便利貼:到貨通知能不能即時一點

現況是「到貨才寄一封 email,使用者隔天才看到」。他要的是**東西一變化,馬上知道**。
訂單狀態更新也是同一類需求:使用者盯著頁面等「出貨了嗎」。

這類需求的共同點是:**變化發生在伺服器那邊,使用者要主動被告知,而不是自己重整頁面去問**。

## 選項一:Polling(輪詢)——最笨但最簡單

前端每隔幾秒就問一次伺服器「有新狀態嗎」:

```
每 5 秒:GET /orders/123/status
```

好處是**簡單到不行**,用一般的 REST API 就做完了,伺服器完全不用改架構。
壞處也很直接:大部分時候答案都是「沒變化」,等於白打了一堆請求;
而且延遲取決於輪詢間隔 —— 5 秒問一次,最壞情況使用者要多等 5 秒才知道。

一個變形是 **長輪詢**:伺服器收到請求後**不馬上回應**,
而是拖著這個連線,等真的有變化(或等到超時)才回傳,收到回應後前端立刻再發下一個請求。
比起死板的定時輪詢,長輪詢延遲更低、白打的請求更少,但每個連線佔用伺服器資源的時間變長了,
是介於 polling 跟真正推播之間的過渡方案。

## 選項二:SSE(Server-Sent Events)——伺服器單向推播

**SSE** 讓伺服器**主動**推送訊息給前端,前端不用再自己問。
它建立在普通 HTTP 之上,瀏覽器內建 `EventSource` API,伺服器端只要維持連線、
不斷寫入 `data: ...\n\n` 格式的訊息就行,不需要額外的協定或函式庫。

限制是**單向**:伺服器能推給前端,前端沒有對稱的管道推回去
(還是得靠一般的 HTTP 請求)。「到貨通知」「訂單狀態變化」這種
只需要伺服器往前端推的場景,SSE 剛好夠用,而且實作簡單、能重用既有的 HTTP 基礎設施。

## 選項三:WebSocket——雙向,協作與聊天才需要

**WebSocket** 建立一條**雙向**的長連線,前端跟伺服器誰都能隨時發訊息給對方。
代價是協定更複雜、需要獨立維護連線狀態,伺服器端的資源管理也更講究
(一條連線要一直開著,量大時要考慮能撐多少併發連線)。

什麼時候真的需要它?當**使用者也要即時發東西給伺服器、而且伺服器要即時轉給其他使用者**時 ——
例如多人協作編輯、即時聊天。到貨通知不需要使用者「回話」給伺服器,用不到 WebSocket 的雙向能力。

## 決策表

| 情境 | 更新頻率 | 方向性 | 連線成本 | 建議 |
|---|---|---|---|---|
| 到貨通知、訂單狀態 | 低頻、非即時關鍵 | 伺服器→前端單向 | 低 | Polling 起步,量大再升 SSE |
| 儀表板數據每幾秒更新 | 中頻 | 伺服器→前端單向 | 中 | SSE |
| 多人協作、即時聊天 | 高頻、雙向互動 | 雙向 | 高 | WebSocket |

## 原則:先用最笨的夠用方案

「好物市集」的到貨通知,使用者能接受幾秒到幾十秒的延遲,不需要毫秒級即時。
**先上 polling(甚至就每分鐘檢查一次),量體與使用者期待真的撐不住了,再往 SSE、WebSocket 升級。**
過早上 WebSocket,換到的是多一套連線管理的維運負擔,卻沒對應到真正的需求。

## 📇 名詞卡

- **長輪詢 (Long Polling)** — 前端發出請求後,伺服器先不回應,拖住連線直到真的有新資料(或逾時)才回傳,是介於定時輪詢與真正推播之間的過渡方案,延遲比定時輪詢低。
  - 想更深可以想想:MDN「HTTP」概念文件中對 long polling 有簡短說明。
- **SSE (Server-Sent Events)** — 建立在普通 HTTP 之上的單向推播機制:伺服器持續往同一條連線寫入訊息,瀏覽器用內建的 EventSource API 接收,不需要額外協定,適合「伺服器推、前端不用回話」的場景。
  - 想更深可以想想:MDN「Server-sent events」條目。
- **WebSocket** — 一條雙向的長連線協定,前端與伺服器都能隨時主動發送訊息,適合聊天、多人協作等需要即時互動的場景,但連線與資源管理比 SSE 複雜。
  - 想更深可以想想:MDN「WebSockets API」條目。
